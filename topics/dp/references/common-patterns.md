# DP Common Patterns

이 문서는 코딩테스트에서 자주 나오는 DP 패턴을 간단히 정리한 문서다.

목표는 패턴을 외우는 것이 아니라, 문제를 봤을 때 "이런 구조인가?"를 빠르게 떠올리는 것이다.

## 1. Linear Accumulation

형태:

- 앞의 몇 개 상태를 이용해 현재 상태를 만든다

예시 감각:

- 계단 오르기
- 연속 선택
- i번째까지 최댓값

포인트:

- `dp[i]` 정의가 명확해야 한다
- 이전 1개, 2개, 또는 몇 개 상태를 참조하는지 확인한다

## 2. Select / Not Select

형태:

- 현재 대상을 선택할지 말지 비교한다

예시 감각:

- 물건 선택
- 인접 요소를 함께 고를 수 없는 경우
- 특정 항목을 쓸지 버릴지 판단

포인트:

- 현재 선택이 다음 상태에 어떤 제약을 만드는지 본다

## 3. Count Ways

형태:

- 경우의 수를 누적한다

예시 감각:

- 도달 방법 수
- 조합 수
- 문자열 구성 방법 수

포인트:

- 최댓값 문제와 섞어 생각하지 않도록 조심한다
- 큰 수 처리나 모듈러 연산이 필요한지 본다

## 4. Sequence Comparison

형태:

- 두 수열 또는 두 문자열을 함께 비교한다

예시 감각:

- 공통 부분 수열
- 문자열 편집
- 두 배열 비교

포인트:

- 보통 `dp[i][j]` 형태가 많이 나온다
- 인덱스 기준과 base case가 중요하다

## 5. Knapsack Style

형태:

- 제한된 자원 안에서 최대 효율 또는 경우의 수를 구한다

예시 감각:

- 무게 제한
- 시간 제한
- 예산 제한

포인트:

- 상태가 "몇 번째까지 봤는지"와 "현재 사용량"을 함께 가질 수 있다
- 1차원 압축이 가능한지도 나중에 검토한다

## 6. Path Counting / Path Optimization

형태:

- 격자나 경로 문제에서 방법 수나 최적값을 누적한다

예시 감각:

- 특정 칸까지 가는 방법 수
- 특정 칸까지의 최소 비용

포인트:

- 이동 방향이 제한되는지
- 장애물이 있는지
- 경로 수인지 최적값인지 먼저 구분한다

## Final Reminder

패턴은 시작점일 뿐이다.

실제 문제를 풀 때는 아래 순서를 따른다.

1. 상태 정의
2. 전이 정의
3. base case 정의
4. 시간복잡도 확인
