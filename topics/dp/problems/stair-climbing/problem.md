# Problem

## Basic Info

- Source: Baekjoon Online Judge
- Problem Number: 2579
- Problem Name: 계단 오르기
- URL: https://www.acmicpc.net/problem/2579
- Primary Topic: dp
- Related Topics: implementation

## Problem Summary

각 계단에는 점수가 있고, 아래에서 시작해서 마지막 계단까지 올라가야 한다.

다만 아래 규칙을 지켜야 한다.

- 한 번에 1칸 또는 2칸 이동할 수 있다
- 연속된 3개의 계단을 모두 밟을 수 없다
- 마지막 계단은 반드시 밟아야 한다

이 조건을 만족하면서 얻을 수 있는 점수의 최댓값을 구하는 문제다.

## Input

- 첫째 줄에 계단 수 `N`이 주어진다
- 다음 `N`개의 줄에 각 계단 점수가 아래에서부터 순서대로 주어진다
- `N`은 300 이하의 자연수다
- 각 계단 점수는 10,000 이하의 자연수다

## Output

- 규칙을 지키며 마지막 계단에 도착했을 때 얻을 수 있는 최대 점수를 출력한다

## Important Signals

- 문제를 읽을 때 중요했던 문장:
  - 마지막 계단은 반드시 밟아야 한다
  - 연속된 세 개의 계단을 모두 밟아서는 안 된다
- 핵심 표현:
  - 최댓값
  - 이전 상태를 이용한 누적 선택
- 구조 해석:
  - 배열 위에서 이전 상태를 기반으로 값을 누적하는 구조

## Constraints That Matter

- 시간복잡도 상한을 추정하게 만든 제약:
  - `N <= 300`이라서 `O(N)` 또는 `O(N^2)` 모두 가능하지만, 이 문제는 더 단순한 `O(N)` DP로 해결할 수 있다
- 메모리 측면에서 주의할 점:
  - 메모리는 크지 않아서 1차원 DP 배열이면 충분하다
- 이 범위에서 불가능한 접근:
  - 완전탐색으로 가능한 경로를 전부 만드는 방식은 불필요하게 비효율적이다
- 이 범위에서 가능한 접근:
  - 1차원 DP
  - 메모이제이션 또는 bottom-up

## First Candidate Algorithms

- 후보 1: DP
- 후보 2: 완전탐색 + 가지치기
- 후보 3: 메모이제이션 DFS

## Early Edge Cases

- 가장 먼저 떠오른 반례:
  - 계단 수가 1개인 경우
  - 계단 수가 2개인 경우
  - 마지막 계단을 반드시 포함해야 해서 앞의 큰 점수를 일부 포기해야 하는 경우
- 경계 조건:
  - `N = 1`
  - `N = 2`
  - `N = 3`

## Problem Point

- 이 문제의 핵심 포인트:
  - `dp[i]`를 “i번째 계단을 반드시 밟고 도착했을 때의 최대 점수”로 정의하는 것
- 조건이 달라지면 바뀌는 풀이:
  - 연속 3계단 금지 조건이 사라지면 단순 누적 방식이나 더 단순한 점화식으로 바뀔 수 있다
  - 이동 규칙이나 선택 제약이 더 복잡해지면 상태 차원이 늘어날 수 있다
