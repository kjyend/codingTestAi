# Notes

## Core Signals

- 문제에서 보이는 핵심 신호:
  - 최댓값을 구한다
  - 현재 선택이 이전 계단 선택 상태에 의존한다
  - 연속 3계단 금지 때문에 단순 그리디로는 어렵다
- 문제에서 가장 중요했던 문장:
  - 연속된 세 개의 계단을 모두 밟아서는 안 된다
  - 마지막 계단은 반드시 밟아야 한다
- 정답 형태:
  - 최댓값

## Candidate Algorithms

- 후보 1: DP
- 후보 2: 완전탐색
- 후보 3: 메모이제이션 DFS

## Final Choice

- 최종 선택한 알고리즘: DP
- 선택 이유:
  - 각 계단에서의 최적 선택이 이전 계단들의 최적 결과에 의존한다
  - 동일한 부분 문제를 반복해서 계산하게 되므로 DP가 자연스럽다
  - `N <= 300`이라 1차원 bottom-up DP로 간단하고 안전하게 구현할 수 있다
- 사용한 자료구조와 이유:
  - `int[] stairScores`
  - `int[] dp`
  - 각 계단 점수와 각 위치까지의 최대 점수를 저장하기에 가장 단순하다

## Complexity Judgment

- 입력 크기 기준 판단:
  - `N`이 300이라 큰 편은 아니지만, 완전탐색으로 경우를 모두 보는 접근은 불필요하다
- Time Complexity:
  - `O(N)`
- Space Complexity:
  - `O(N)`
- 시간초과 가능성이 있었던 접근:
  - 가능한 이동 경로를 재귀적으로 전부 탐색하는 방식

## Problem Point

- 이 문제의 핵심 포인트:
  - `dp[i] = i번째 계단을 반드시 밟고 도착했을 때의 최대 점수`
  - 현재 계단 `i`에 도달하는 경우는 크게 두 가지다
    - `i-2`에서 바로 오는 경우
    - `i-3`에서 `i-1`을 밟고 오는 경우
- 이 문제를 풀 때 꼭 봐야 했던 조건:
  - 마지막 계단을 반드시 밟아야 한다
  - 연속 3계단을 모두 밟으면 안 된다

## If Conditions Change

- 조건이 달라졌다면 사용할 수 있는 다른 알고리즘:
  - 연속 3계단 금지 조건이 더 복잡해지면 상태를 하나 더 둔 DP
  - 입력이 아주 작으면 완전탐색도 가능
- 지금 풀이가 더 이상 맞지 않게 되는 조건:
  - 한 번에 오를 수 있는 칸 수가 여러 종류로 늘어나고, 연속 선택 제약도 복합적으로 바뀌는 경우

## Rejected Ideas

- 시도했지만 버린 접근:
  - 현재 계단 점수만 보고 큰 값을 택하는 그리디
  - 모든 이동 경로를 재귀로 탐색
- 버린 이유:
  - 그리디는 마지막 계단 포함과 연속 3계단 금지 조건을 동시에 만족시키는 전역 최적을 보장하지 않는다
  - 완전탐색은 불필요하게 많은 중복 계산이 발생한다

## Test And Pitfalls

- 예제 테스트:
  - 문제 예제 입력에서 정답 `75`가 나오는지 확인
- 반례 테스트:
  - `N = 3`에서 세 계단을 모두 밟으면 안 되는 상황 확인
- 경계 조건 테스트:
  - `N = 1`, `N = 2`
- 실수 포인트:
  - `N = 1`, `N = 2`, `N = 3` base case를 따로 처리하지 않으면 인덱스 오류나 잘못된 점화식이 나올 수 있다
  - `dp[i - 1]`를 그대로 쓰면 연속 3계단 금지 조건을 깨는 경로가 섞일 수 있다

## Code Comment Points

- 코드에서 설명이 꼭 필요한 부분:
  - `dp[i]`의 의미
  - `dp[2]`를 왜 저렇게 초기화하는지
  - 반복문에서 왜 두 경우만 비교하면 되는지
  - 마지막 값을 그대로 반환해도 되는 이유
- 상태 의미로 주석을 남길 부분:
  - `dp[i] = i번째 계단을 반드시 밟고 도착했을 때의 최대 점수`
- 점화식 또는 핵심 전이로 주석을 남길 부분:
  - `i-2`에서 바로 오는 경우
  - `i-3 -> i-1 -> i`로 오는 경우
- base case 또는 함정 처리로 주석을 남길 부분:
  - `N <= 2`를 따로 처리하는 이유
  - `N = 3`에서 세 계단을 모두 밟을 수 없다는 점

## Review

- 다시 풀면 더 단순하게 만들 수 있는 부분:
  - `N <= 2`는 별도 처리하고, `N >= 3`부터 일관된 점화식을 적용하면 깔끔하다
- 다음에 비슷한 문제가 나오면 떠올릴 신호:
  - “현재 칸을 밟는 조건부 최대값”
  - “연속 선택 금지”
  - “마지막 위치는 반드시 포함”
