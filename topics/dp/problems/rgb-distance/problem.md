# Problem

## Basic Info

- Source: Baekjoon Online Judge
- Problem Number: 1149
- Problem Name: RGB거리
- URL: https://www.acmicpc.net/problem/1149
- Primary Topic: dp
- Related Topics: implementation

## Problem Summary

RGB거리에는 집이 `N`개 있고, 집은 순서대로 한 줄에 놓여 있다.

각 집은 빨강, 초록, 파랑 중 하나의 색으로 칠해야 하고, 집마다 각 색으로 칠하는 비용이 다르다.

다만 인접한 두 집은 같은 색으로 칠할 수 없다.

이 조건을 만족하면서 모든 집을 칠하는 최소 비용을 구하는 문제다.

## Input

- 첫째 줄에 집의 수 `N`이 주어진다
- 둘째 줄부터 `N`개의 줄에 각 집을 빨강, 초록, 파랑으로 칠하는 비용이 주어진다
- `N`은 2 이상 1,000 이하이다
- 각 비용은 1,000 이하의 자연수다

## Output

- 모든 집을 칠하는 최소 비용을 출력한다

## Important Signals

- 문제를 읽을 때 중요했던 문장:
  - 인접한 집은 같은 색으로 칠할 수 없다
  - 최소 비용을 구한다
- 핵심 표현:
  - 최소값
  - 현재 선택이 이전 집의 선택에 의존한다
- 구조 해석:
  - `집 번호 x 현재 색상` 상태로 최소 비용을 누적하는 구조

## Constraints That Matter

- 시간복잡도 상한을 추정하게 만든 제약:
  - `N <= 1000`이라 각 집마다 색 3개를 확인하는 `O(N)` DP가 적절하다
- 메모리 측면에서 주의할 점:
  - 집 수와 색 수가 작아서 `N x 3` DP 배열이면 충분하다
- 이 범위에서 불가능한 접근:
  - 모든 색칠 조합을 전부 탐색하는 방식
- 이 범위에서 가능한 접근:
  - 2차원 DP
  - 입력 배열을 직접 갱신하는 in-place DP

## First Candidate Algorithms

- 후보 1: 2차원 DP
- 후보 2: 입력 배열 직접 갱신 방식의 DP
- 후보 3: 완전탐색

## Early Edge Cases

- 가장 먼저 떠오른 반례:
  - 집 수가 작을 때도 인접한 집 색이 달라야 한다는 조건
  - 현재 색 비용이 가장 작더라도 이전 집과 같은 색이면 선택할 수 없는 경우
  - 마지막 집의 색이 특정 색으로 끝나야 하는 것이 아니라 세 색 중 최소여야 하는 경우
- 경계 조건:
  - `N = 2`
  - 모든 색 비용이 같은 경우
  - 한 색이 매번 가장 싸지만 연속으로 선택할 수 없는 경우

## Problem Point

- 이 문제의 핵심 포인트:
  - `dp[i][color] = i번째 집까지 칠했을 때, i번째 집의 색이 color인 최소 비용`
- 조건이 달라지면 바뀌는 풀이:
  - 인접 제약이 더 길게 이어지면 상태에 이전 색 정보가 더 필요할 수 있다
  - 최소 비용이 아니라 경우의 수를 묻는 문제라면 합산형 DP로 바뀐다
