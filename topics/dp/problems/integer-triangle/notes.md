# Notes

## Core Signals

- 문제에서 보이는 핵심 신호:
  - 최댓값을 구한다
  - 현재 칸의 답이 이전 층의 칸들에 의존한다
  - 같은 위치까지의 최대 합을 반복해서 재사용할 수 있다
- 문제에서 가장 중요했던 문장:
  - 현재 층에서 선택된 수의 대각선 왼쪽 또는 오른쪽에 있는 것만 선택할 수 있다
- 정답 형태:
  - 최댓값

## Candidate Algorithms

- 후보 1: 2차원 DP
- 후보 2: 입력 배열 갱신 방식의 DP
- 후보 3: 완전탐색

## Final Choice

- 최종 선택한 알고리즘: 2차원 bottom-up DP
- 선택 이유:
  - 각 칸까지의 최대 합이 이전 층의 최대 합 두 개 중 큰 값에 현재 값을 더하는 형태로 계산된다
  - `n <= 500`이라 `O(n^2)` DP가 충분히 가능하다
  - 상태 의미를 분리한 별도 DP 배열이 학습용으로 더 읽기 쉽다
- 사용한 자료구조와 이유:
  - `int[][] triangle`
  - `int[][] dp`
  - 삼각형 원본과 최대 합 상태를 분리해 이해하기 쉽게 유지한다

## Complexity Judgment

- 입력 크기 기준 판단:
  - 각 층의 칸 수를 모두 합치면 대략 `O(n^2)` 개 상태가 나온다
- Time Complexity:
  - `O(n^2)`
- Space Complexity:
  - `O(n^2)`
- 시간초과 가능성이 있었던 접근:
  - 모든 경로를 재귀로 직접 만들어 보는 방식

## Problem Point

- 이 문제의 핵심 포인트:
  - `dp[row][column] = (row, column)에 도착했을 때의 최대 합`
  - 왼쪽 가장자리와 오른쪽 가장자리는 부모가 하나뿐이라 별도 처리해야 한다
  - 가운데 칸만 부모 두 개 중 큰 값을 선택하면 된다
- 이 문제를 풀 때 꼭 봐야 했던 조건:
  - 아래로 내려갈 때 선택 가능한 위치가 두 곳으로 제한된다
  - 가장자리 칸은 부모가 하나뿐이다

## If Conditions Change

- 조건이 달라졌다면 사용할 수 있는 다른 알고리즘:
  - 경우의 수 문제라면 합산형 DP
  - 이동 가능한 방향이 더 많아지면 전이 후보가 늘어난 DP
- 지금 풀이가 더 이상 맞지 않게 되는 조건:
  - 이동 규칙이 층 단위가 아니라 임의 그래프처럼 바뀌는 경우

## Rejected Ideas

- 시도했지만 버린 접근:
  - 모든 경로를 DFS로 탐색
  - 매 단계에서 더 큰 값만 고르는 그리디
- 버린 이유:
  - DFS는 경로 수가 너무 많아진다
  - 그리디는 당장 큰 값을 골라도 전체 최댓값을 보장하지 않는다

## Test And Pitfalls

- 예제 테스트:
  - 문제 예제 입력에서 정답 `30`이 나오는지 확인
- 반례 테스트:
  - 중간 값보다 가장자리 누적이 더 큰 경로를 선택해야 하는 상황 확인
- 경계 조건 테스트:
  - `n = 1`
  - 모든 값이 동일한 큰 입력
- 실수 포인트:
  - 가장자리 칸에도 부모가 두 개 있다고 착각하는 실수
  - 마지막 줄의 최댓값을 구해야 하는데 마지막 칸만 반환하는 실수

## Code Comment Points

- 코드에서 설명이 꼭 필요한 부분:
  - `dp[row][column]`의 의미
  - 가장자리 칸을 따로 처리하는 이유
  - 가운데 칸에서 두 부모 중 큰 값을 택하는 부분
  - 마지막 줄 전체에서 최댓값을 다시 구하는 이유
- 상태 의미로 주석을 남길 부분:
  - `dp[row][column] = 해당 칸에 도착했을 때의 최대 합`
- 점화식 또는 핵심 전이로 주석을 남길 부분:
  - 왼쪽 가장자리 / 오른쪽 가장자리 / 가운데 칸의 전이 차이
- base case 또는 함정 처리로 주석을 남길 부분:
  - `dp[0][0]` 초기화
  - 마지막 값이 아니라 마지막 줄의 최대값을 답으로 써야 한다는 점
