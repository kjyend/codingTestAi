# Problem

## Basic Info

- Source: Baekjoon Online Judge
- Problem Number: 1932
- Problem Name: 정수 삼각형
- URL: https://www.acmicpc.net/problem/1932
- Primary Topic: dp
- Related Topics: implementation

## Problem Summary

맨 위 숫자에서 시작해서 아래층으로 내려가며 숫자를 하나씩 선택한다.

현재 위치에서 내려갈 수 있는 곳은 다음 층의 대각선 왼쪽 또는 대각선 오른쪽뿐이다.

이렇게 맨 아래층까지 내려갈 때, 선택한 숫자의 합이 최대가 되는 경로의 합을 구하는 문제다.

## Input

- 첫째 줄에 삼각형의 크기 `n`이 주어진다
- 둘째 줄부터 `n + 1`번째 줄까지 정수 삼각형이 주어진다
- `n`은 1 이상 500 이하이다
- 삼각형의 각 수는 0 이상 9999 이하의 정수다

## Output

- 합이 최대가 되는 경로에 있는 수의 합을 출력한다

## Important Signals

- 문제를 읽을 때 중요했던 문장:
  - 아래층에서는 현재 위치의 대각선 왼쪽 또는 오른쪽만 선택할 수 있다
  - 합이 최대가 되는 경로를 구한다
- 핵심 표현:
  - 최댓값
  - 이전 층의 결과를 이용한 누적 선택
- 구조 해석:
  - 2차원 삼각형 위에서 각 칸까지의 최대 합을 누적하는 구조

## Constraints That Matter

- 시간복잡도 상한을 추정하게 만든 제약:
  - `n <= 500`이라 완전탐색으로 모든 경로를 보는 것은 불가능하고, `O(n^2)` DP가 적절하다
- 메모리 측면에서 주의할 점:
  - 삼각형 전체를 저장해도 충분하지만, DP 배열 크기 계산은 명확히 해야 한다
- 이 범위에서 불가능한 접근:
  - 모든 경로를 DFS로 탐색하는 방식
- 이 범위에서 가능한 접근:
  - 2차원 bottom-up DP
  - 입력 배열을 직접 갱신하는 in-place DP

## First Candidate Algorithms

- 후보 1: 2차원 DP
- 후보 2: 입력 배열을 직접 갱신하는 DP
- 후보 3: 완전탐색

## Early Edge Cases

- 가장 먼저 떠오른 반례:
  - `n = 1`인 경우
  - 왼쪽 가장자리와 오른쪽 가장자리를 갱신할 때 부모가 하나뿐인 경우
  - 중간 칸은 부모가 둘인데 가장자리는 하나뿐이라는 점을 놓치는 경우
- 경계 조건:
  - `n = 1`
  - 삼각형의 모든 값이 같은 경우
  - 가장 큰 합이 오른쪽 가장자리로만 이어지는 경우

## Problem Point

- 이 문제의 핵심 포인트:
  - `dp[row][column] = 해당 칸에 도착했을 때 만들 수 있는 최대 합`
- 조건이 달라지면 바뀌는 풀이:
  - 이동 규칙이 더 복잡해지면 상태 정의가 달라질 수 있다
  - 최댓값이 아니라 경우의 수를 구하면 같은 구조라도 합산형 DP로 바뀐다
