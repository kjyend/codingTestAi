# Notes

## Classification

- Primary Topic: graph
- Problem Category: traversal
- Primary Algorithm: bfs

## Core Signals

- 문제에서 보이는 핵심 신호:
  - 격자 구조
  - 최단 거리
  - 상하좌우 이동
  - 가중치 없음
- 문제에서 가장 중요했던 문장:
  - 지나야 하는 최소 칸 수를 구한다
- 정답 형태:
  - 최솟값

## Candidate Algorithms

- 후보 1: BFS
- 후보 2: DFS + 완전탐색
- 후보 3: 다익스트라

## Final Choice

- 최종 선택한 알고리즘: BFS
- 선택 이유:
  - 가중치가 없는 그래프에서 최단 거리를 구하는 대표 알고리즘이다
  - 격자 각 칸을 정점으로 보면 BFS로 처음 도착하는 거리가 곧 최소 거리다
  - `N * M` 범위가 작아서 큐 기반 탐색으로 충분하다
- 사용한 자료구조와 이유:
  - `char[][] maze`
  - `boolean[][] visited`
  - `int[][] distance`
  - `ArrayDeque<int[]>`
  - 격자, 방문 여부, 거리, BFS 큐를 분리해 의미를 명확히 유지한다

## Complexity Judgment

- 입력 크기 기준 판단:
  - 최대 10,000칸이라 각 칸을 한 번씩 방문하는 BFS가 적절하다
- Time Complexity:
  - `O(N * M)`
- Space Complexity:
  - `O(N * M)`
- 시간초과 가능성이 있었던 접근:
  - 도착점까지 가능한 경로를 재귀로 모두 탐색하는 방식

## Problem Point

- 이 문제의 핵심 포인트:
  - 격자 각 칸을 정점으로 보고, 이동 가능한 상하좌우 칸을 간선으로 보는 것
  - 가중치가 없으므로 BFS 거리 배열이 곧 최단 거리다
- 이 문제를 풀 때 꼭 봐야 했던 조건:
  - 이동 가능한 칸은 `1`
  - 시작 칸도 지나간 칸 수에 포함된다
  - 방문 체크를 적절한 시점에 해야 중복 탐색을 막을 수 있다

## If Conditions Change

- 조건이 달라졌다면 사용할 수 있는 다른 알고리즘:
  - 이동 비용이 생기면 다익스트라
  - 단순 연결 여부만 묻는다면 DFS도 가능
- 지금 풀이가 더 이상 맞지 않게 되는 조건:
  - 칸마다 이동 비용이 다르거나, 가중치가 추가되는 경우

## Rejected Ideas

- 시도했지만 버린 접근:
  - DFS로 모든 경로 탐색
  - 다익스트라
- 버린 이유:
  - DFS는 최단 거리 보장이 없고, 모든 경로를 보면 비효율적이다
  - 다익스트라는 가중치가 있을 때 필요하고, 이 문제는 BFS가 더 단순하다

## Test And Pitfalls

- 예제 테스트:
  - 공식 예제 입력에서 정답 `15`가 나오는지 확인
- 반례 테스트:
  - 직선처럼 한 방향으로만 이동 가능한 미로에서 거리 계산 확인
- 경계 조건 테스트:
  - 작은 격자
  - 전부 `1`인 격자
- 실수 포인트:
  - 시작 칸 거리를 `0`으로 둘지 `1`로 둘지 문제 조건과 맞추지 않는 실수
  - 방문 체크를 큐에서 꺼낼 때 해서 중복 삽입이 생기는 실수
  - `0` 칸을 이동 가능한 칸으로 잘못 해석하는 실수

## Code Comment Points

- 코드에서 설명이 꼭 필요한 부분:
  - 격자 각 칸을 그래프 정점처럼 본다는 점
  - BFS에서 처음 도착한 거리가 최단 거리라는 점
  - 시작 칸 거리 초기화 이유
- 상태 의미로 주석을 남길 부분:
  - `distance[row][column] = 시작점에서 해당 칸까지의 최소 칸 수`
- 점화식 또는 핵심 전이로 주석을 남길 부분:
  - 상하좌우 이동
  - 큐에 넣을 때 방문 처리하는 이유
- base case 또는 함정 처리로 주석을 남길 부분:
  - 시작 칸 거리 `1`
  - 범위 밖, 벽, 방문한 칸 제외 처리
