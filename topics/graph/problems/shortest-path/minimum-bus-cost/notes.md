# Notes

## Classification

- Primary Topic: graph
- Problem Category: shortest-path
- Primary Algorithm: dijkstra

## Core Signals

- 문제에서 보이는 핵심 신호:
  - 최소 비용
  - 가중치 있는 방향 그래프
  - 단일 시작점 최단거리
- 문제에서 가장 중요했던 문장:
  - 출발 도시에서 도착 도시까지 가는 최소 비용을 구한다
- 정답 형태:
  - 최솟값

## Candidate Algorithms

- 후보 1: 다익스트라
- 후보 2: 벨만포드
- 후보 3: DFS + 완전탐색

## Final Choice

- 최종 선택한 알고리즘: 다익스트라
- 선택 이유:
  - 간선 가중치가 모두 0 이상이므로 다익스트라 적용 조건에 맞는다
  - 특정 시작점에서 특정 도착점까지의 최소 비용을 효율적으로 구할 수 있다
  - `M <= 100,000`이라 인접 리스트와 우선순위 큐를 쓰는 구현이 적절하다
- 사용한 자료구조와 이유:
  - `List<Edge>[] graph`
  - `int[] distances`
  - `PriorityQueue<State>`
  - 간선 목록, 최단거리 배열, 현재 후보 정점을 효율적으로 관리하기 좋다

## Complexity Judgment

- 입력 크기 기준 판단:
  - 간선 수가 많아서 모든 정점을 매번 선형 탐색하는 방식보다 우선순위 큐가 유리하다
- Time Complexity:
  - `O((V + E) log V)` 또는 실무적으로 `O(E log V)`로 본다
- Space Complexity:
  - `O(V + E)`
- 시간초과 가능성이 있었던 접근:
  - 모든 경로를 직접 탐색하는 방식
  - 우선순위 큐 없이 비효율적으로 최단 정점을 찾는 방식

## Problem Point

- 이 문제의 핵심 포인트:
  - `distances[city] = 시작 도시에서 city까지의 현재까지 알려진 최소 비용`
  - 우선순위 큐에서 현재 비용이 가장 작은 도시를 먼저 꺼내며 거리 갱신을 진행한다
  - 더 좋은 경로가 발견되면 큐에 다시 넣고, 오래된 상태는 건너뛴다
- 이 문제를 풀 때 꼭 봐야 했던 조건:
  - 비용이 음수가 아니다
  - 도착 가능성이 항상 보장된다
  - 같은 출발/도착 도시 쌍의 버스가 여러 개 있을 수 있다

## If Conditions Change

- 조건이 달라졌다면 사용할 수 있는 다른 알고리즘:
  - 음수 간선이 있으면 벨만포드
  - 모든 정점 쌍이면 플로이드 워셜
- 지금 풀이가 더 이상 맞지 않게 되는 조건:
  - 음수 가중치가 추가되는 경우

## Rejected Ideas

- 시도했지만 버린 접근:
  - 벨만포드
  - DFS로 모든 경로 탐색
- 버린 이유:
  - 벨만포드는 가능하지만 이 문제에서는 다익스트라가 더 효율적이다
  - DFS는 경로 수가 너무 많아진다

## Test And Pitfalls

- 예제 테스트:
  - 공식 예제 입력에서 정답 `4`가 나오는지 확인
- 반례 테스트:
  - 직접 가는 비용보다 중간 정점을 거치는 비용이 더 작은 경우
  - 같은 도시쌍 간선이 여러 개인 경우
- 경계 조건 테스트:
  - 비용이 0인 간선이 포함된 경우
  - 작은 정점 수와 큰 간선 수 조합
- 실수 포인트:
  - 우선순위 큐에서 나온 오래된 상태를 걸러내지 않는 실수
  - 방향 그래프인데 양방향처럼 간선을 추가하는 실수
  - 도착 도시를 바로 만나도 더 싼 경로 가능성을 고려하지 않고 조기 종료하는 실수

## Code Comment Points

- 코드에서 설명이 꼭 필요한 부분:
  - 도시와 버스 정보를 가중치 있는 방향 그래프로 보는 점
  - `distances` 배열의 의미
  - 우선순위 큐에서 오래된 상태를 건너뛰는 이유
- 상태 의미로 주석을 남길 부분:
  - `distances[city] = 시작 도시에서 city까지의 최소 비용`
- 점화식 또는 핵심 전이로 주석을 남길 부분:
  - 현재 도시를 거쳐 인접 도시 비용을 갱신하는 부분
- base case 또는 함정 처리로 주석을 남길 부분:
  - 시작 도시 거리를 0으로 초기화하는 부분
  - 큐에서 나온 비용이 이미 더 크면 건너뛰는 부분
