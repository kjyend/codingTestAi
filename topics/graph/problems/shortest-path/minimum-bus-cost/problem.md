# Problem

## Basic Info

- Source: Baekjoon Online Judge
- Problem Number: 1916
- Problem Name: 최소비용 구하기
- URL: https://www.acmicpc.net/problem/1916
- Primary Topic: graph
- Related Topics: shortest-path
- Problem Category: shortest-path
- Primary Algorithm: dijkstra

## Problem Summary

`N`개의 도시와 `M`개의 버스 정보가 주어진다.

각 버스는 출발 도시, 도착 도시, 비용을 가진다.

특정 출발 도시에서 특정 도착 도시까지 가는 최소 비용을 구하는 문제다.

## Input

- 첫째 줄에 도시의 개수 `N`이 주어진다
- 둘째 줄에 버스의 개수 `M`이 주어진다
- 다음 `M`개의 줄에 `출발 도시`, `도착 도시`, `비용`이 주어진다
- 마지막 줄에 `출발 도시`, `도착 도시`가 주어진다
- `N`은 1 이상 1,000 이하이다
- `M`은 1 이상 100,000 이하이다
- 버스 비용은 0 이상 100,000 미만이다
- 출발점에서 도착점을 갈 수 있는 경우만 입력으로 주어진다

## Output

- 출발 도시에서 도착 도시까지 가는 최소 비용을 출력한다

## Important Signals

- 문제를 읽을 때 중요했던 문장:
  - 비용이 있는 간선이다
  - 최소 비용을 구한다
  - 특정 시작점에서 특정 도착점까지의 최단거리를 구한다
- 핵심 표현:
  - 최소 비용
  - 가중치 있는 그래프
  - 시작점과 도착점이 주어짐
- 구조 해석:
  - 도시를 정점, 버스를 가중치 있는 방향 간선으로 보는 그래프

## Constraints That Matter

- 시간복잡도 상한을 추정하게 만든 제약:
  - `M <= 100,000`이라 단순한 `O(N^2)`보다 우선순위 큐 기반 다익스트라가 더 적절하다
- 메모리 측면에서 주의할 점:
  - 인접 리스트와 거리 배열이면 충분하다
- 이 범위에서 불가능한 접근:
  - 모든 경로를 DFS로 탐색하는 방식
  - 인접 행렬 기반의 큰 메모리 사용
- 이 범위에서 가능한 접근:
  - 다익스트라
  - 인접 리스트 + 우선순위 큐

## First Candidate Algorithms

- 후보 1: 다익스트라
- 후보 2: 벨만포드
- 후보 3: DFS + 완전탐색

## Early Edge Cases

- 가장 먼저 떠오른 반례:
  - 같은 두 도시 사이에 여러 버스가 있는 경우
  - 직접 가는 경로보다 중간 도시를 거치는 경로가 더 싼 경우
  - 비용이 0인 버스가 있는 경우
- 경계 조건:
  - 도시 수가 작은 경우
  - 버스 수가 매우 많은 경우
  - 최단거리가 여러 경로로 같은 경우

## Problem Point

- 이 문제의 핵심 포인트:
  - 가중치가 있는 단일 시작점 최단거리 문제이므로 다익스트라가 가장 자연스럽다
- 조건이 달라지면 바뀌는 풀이:
  - 음수 간선이 생기면 벨만포드를 봐야 한다
  - 모든 정점 쌍 최단거리가 필요하면 플로이드 워셜을 볼 수 있다
