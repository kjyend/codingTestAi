# Graph Modeling Checklist

그래프 문제는 구현보다 먼저 모델링이 중요하다.

이 문서는 문제를 정점과 간선으로 바꾸는 체크리스트다.

## First Question

먼저 아래 두 문장을 말할 수 있어야 한다.

- 정점은 무엇인가
- 간선은 무엇인가

예:

- 사람 = 정점, 친구 관계 = 간선
- 칸 = 정점, 상하좌우 이동 가능 = 간선
- 상태 하나 = 정점, 연산 하나 = 간선

## Checklist

1. 정점이 실제 객체인지, 상태인지
2. 간선이 양방향인지 단방향인지
3. 간선에 가중치가 있는지
4. 방문 체크가 필요한지
5. 중복 상태가 생기는지

## Common Shapes

### 1. Explicit Graph

- 정점과 간선 정보가 직접 주어지는 경우

### 2. Grid Graph

- 격자의 각 칸을 정점으로 보는 경우

### 3. State Graph

- 위치, 숫자, 문자열 상태를 정점으로 보는 경우

## Warning Signs

- 정점과 간선을 한 문장으로 설명하지 못한다
- 방문 배열이 무엇을 의미하는지 애매하다
- 가중치 유무를 끝까지 헷갈린다
- BFS와 다익스트라 후보를 구분하지 못한다

## Practical Rule

모델링이 흔들리면 코드를 바로 쓰지 않는다.

먼저 `notes.md`에 아래를 적는다.

- 정점
- 간선
- 방향성
- 가중치
- 시작점 / 종료점
