# Graph Pitfalls

이 문서는 graph 문제를 풀 때 자주 하는 실수를 정리한 문서다.

## 1. 그래프로 보지 못하는 실수

배열 문제처럼 보이지만 실제로는 연결 관계 문제인 경우가 많다.

## 2. 방문 처리 시점 오류

큐에 넣을 때 방문 처리할지, 꺼낼 때 처리할지에 따라 중복 탐색이 생길 수 있다.

## 3. 방향성 실수

무방향 그래프인데 한쪽만 간선을 넣거나, 방향 그래프인데 양방향으로 넣는 실수가 많다.

## 4. 가중치 유무를 놓치는 실수

최단 거리라고 무조건 BFS가 아니다.

## 5. 상태 그래프를 과소평가하는 실수

문자열, 숫자, 위치 상태도 그래프 노드가 될 수 있다.

## Final Check

코드를 쓰기 전에 아래를 확인한다.

1. 정점과 간선을 한 문장으로 설명할 수 있는가
2. 방문 배열 의미가 명확한가
3. 가중치 유무를 확인했는가
4. 종료 조건이 분명한가
